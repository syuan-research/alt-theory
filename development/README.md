# Development

This directory contains public engineering material that helps contributors
and coding agents understand the implementation. It is not an active process
workspace and should not accumulate private plans, status logs, handoffs, or
agent-session output.

## Structure

- `architecture/` — current technical system maps.
- `features/backend/` and `features/frontend/` — retained design and acceptance
  evidence for implemented features.
- `issues/` — retained analysis or fix notes with continuing engineering value.
- `compound/` — existing decisions, explorations, and research notes. This is a
  transitional retained collection; later curation may select, rename, or
  remove material without blocking the first public release.
- `releases/` — version-specific testing, bundle, and release material that is
  not yet part of the future public documentation system.

Start with `architecture/core-session-engine.md` for runtime behavior,
`architecture/researcher-console.md` for the application UI, and
`architecture/local-windows-bundle.md` for local packaging.

New user/pro-user documentation should wait for the dedicated docs information
architecture work rather than being placed here by default.
