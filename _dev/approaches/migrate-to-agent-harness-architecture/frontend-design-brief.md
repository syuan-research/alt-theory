## 1. 布局结构

标准 chatbot 三栏布局：

```
┌──────────────┬─────────────────────────────┐
│  左侧边栏     │         主聊天区域            │
│  (240px)     │                             │
│              │  ┌─────────────────────────┐│
│  会话历史     │  │     消息列表              ││
│  ──────────  │  │  (可滚动，从上到下)        ││
│  设置面板     │  │                         ││
│  - KB 选择   │  │                         ││
│  - Profile   │  └─────────────────────────┘│
│  - 模型       │  ┌─────────────────────────┐│
│              │  │     输入框 + 发送按钮      ││
│              │  └─────────────────────────┘│
└──────────────┴─────────────────────────────┘
```

**关键点**：
- 左侧边栏包含两个区域：会话历史（上）+ 设置面板（下）
- 设置面板包含 KB 选择器、Profile 选择器（MVP 阶段可折叠）
- 主区域是标准聊天界面

## 2. 消息渲染

### 2.1 消息类型

| 消息类型 | 渲染方式 |
|----------|----------|
| 用户消息 | 右对齐气泡，纯文本 |
| Agent 文字回复 | 左对齐，Markdown 渲染（标题、列表、代码块、表格） |
| Agent 推理过程 | 折叠显示（默认收起），小字灰色，用户可展开 |
| 工具调用 | 小 spinner + 固定文案（"正在搜索知识库..."），不显示命令细节 |
| 工具完成 | 小勾 + "完成"，不显示原始输出 |

### 2.2 流式渲染

Agent 回复是流式的（逐字出现）。需要实现：
- 文字逐字追加（不是等全部完成再显示）
- 流式过程中 Markdown 实时渲染（可以用 marked.js）
- 输入框在 agent 回复期间禁用，显示"停止"按钮

## 3. 左侧边栏详细设计

### 3.1 会话历史

- 列表显示历史会话（每条显示：标题/首句 + 时间）
- 点击切换会话（MVP 阶段：切换 = 新 session，历史只读）
- "新对话"按钮在顶部
- MVP 阶段：会话历史可以只显示当前 session，不做持久化列表

### 3.2 设置面板

```
知识库
[ep-core ▼]          ← dropdown

用户档案
[default ▼]          ← dropdown

[应用设置]           ← 切换后创建新对话（需提示用户）
```

**切换提示**：切换 KB 或 Profile 会开始新对话，需要一个确认提示（"切换知识库将开始新对话，确认？"）。

## 4. 工具调用的视觉处理

这是 Alt Theory 与普通 chatbot 的主要差异。建议：

- **不显示**：工具命令（`grep -r "ART" ./kb`）、原始输出（文件内容）
- **显示**：工具类型对应的人类可读文案
  - `read` → "正在读取知识库..."
  - `grep` → "正在搜索相关理论..."
  - `find` → "正在查找文件..."
  - `bash` → "正在执行操作..."（如果未来启用）
- **显示**：agent 在工具调用前后的推理文字（这是用户能理解的部分）

---

## 5. MVP 范围建议

**必须有**：
- 左侧边栏（会话历史 + KB/Profile 选择器）
- 消息列表（用户消息 + agent 回复，Markdown 渲染）
- 流式文字渲染
- 工具调用状态显示（spinner + 文案，不显示细节）
- 输入框 + 发送/停止按钮

**可以没有**（MVP 后）：
- 会话历史持久化列表（MVP 只显示当前 session）
- 会话标题自动生成
- 消息导出
- 深色模式

---

## 6. 技术选型建议

| 需求 | 建议方案 |
|------|----------|
| Markdown 渲染 | marked.js + DOMPurify（防 XSS） |
| 代码高亮 | highlight.js（按需加载） |
| 流式文字 | 直接 DOM 操作（不需要框架） |
| 样式 | Tailwind CSS（utility-first，快速原型） |
| 框架 | 无框架（vanilla JS）或轻量框架（Preact） |

**不建议用 React/Vue**：MVP 阶段引入框架会增加构建复杂度，vanilla JS 足够。
