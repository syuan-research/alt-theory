    为什么标题中的关键词比正文中的更有价值

    你说得对，"Directed Attention" 在正文任何位置都能 grep 到。区别在于：

    效率 — 搜索-阅读的步骤数。

    场景A：正文中的 "Directed Attention"
    1. grep "Directed Attention" → 命中 47 个结果
    2. 逐个读上下文判断哪个是定义、哪个是引用、哪个是对比
    3. 找到目标 → 可能读了 5-8 个片段

    场景B：标题中的 "Directed Attention"
    1. grep "##.*Directed Attention" → 命中 1-2 个结果
    2. 直接精确定位到概念定义段落
    3. 完成

    标题 = 已经被人类（或上一轮 AI）标注好的语义摘要。 grep 命中标题意味着精确度更高、需要的后续阅读更少。

    offset 参数是什么

    offset 是 Read 工具的参数，意思是"从第 N 行开始读"。它不是"块"，是行号偏移。

    Read(file, offset=50, limit=30)
    → 读第 50-80 行

    用途：当 grep 返回"匹配在第 47 行"时，我可以 offset=40, limit=30 读取 40-70 行，获取匹配点的上下文。

    如果你的文档有 500 行，我不用读全部，可以精确读取需要的部分。这就是"标题重要"的另一个原因 —
    标题给了我行号锚点，我可以用 offset 精准跳转。

    对你的 KB note 设计的实际建议

    最有用的格式：
    ---
    theory: Attention Restoration Theory
    author: Kaplan
    year: 1995
    ---

    # Attention Restoration Theory (Kaplan, 1995)

    ## Summary
    [一段自然文本]

    ## Concept: Directed Attention
    [定义和解释]

    ## Concept: Directed Attention Fatigue
    [定义和解释]

    ## Mechanism: Four Components of Restoration
    [机制描述]

    ## Boundary Conditions
    [适用范围和限制]

    ## Related Theories
    - [[Stress Recovery Theory (Ulrich, 1991)]]
    - [[Environmental Preference Matrix (Kaplan, 1987)]]

    ## Source
    - Original paper: `./papers/Kaplan_1995_Restorative_Benefits.pdf`

    为什么这个格式对我好用：
    1. YAML frontmatter → 我能快速 grep 元数据
    2. ## Concept: X → 我能精确定位到特定概念（grep + offset）
    3. [[wikilink]] → 我能 grep 这些链接发现理论关联
    4. Source: path → 需要查原文时我能 Read PDF
    5. 自然段落 → 嵌入向量质量更好（语义完整）
    6. 同一个文件同时服务 grep 和 RAG — 不需要两种格式
