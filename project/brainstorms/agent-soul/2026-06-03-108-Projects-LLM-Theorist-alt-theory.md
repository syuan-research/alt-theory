# Flomo Memo 2026-06-03

让我来想一想，对于要做的 agents.MD 的系统，这个系统提示词可以怎么写。可以仿照一下 Openclaw 里面的一些东西，比如说他是一个特别有研究者气质的人，然后他并不坚定，所以他知道他所接触的现象和知识是在一个置信程度的灰度上的，而不是极端的。同时他是一个 mentor，他理解人们可以拥有各种各样的观点和对知识的认识，而他只会稍稍地给他们呈现一些也许不同的东西，但他并不致力于说服别人。同时他也绝对不会谄媚，因为他有自己的观点，对事情的置信程度有自己的认识，他有各种见解。当人们拿尖锐的关于什么是对错的问题来问他时，他有很多在高维度的解释范围，比如说这个世界上有很多有效的东西，它们是有关联的，它们其实是很多在特定的 context、特定的语境下、特定的文脉中，一些变量组合才会有效的。你会想这只是其中一个例子了。

这一段我可能想的也不是很成熟，但是我觉得可以把一些常见的尺度、个人的角度这些东西放进去，就是真正有作用关系的东西，其实就像是一个耸立的几座拟合度的山峰一样的感觉，而大家总是往往以为这个山峰这些参数的组合可以有更多的可能性。就这一段，我得去找到我之前写的笔记的原文去核实。我觉得这一段应该给这个系统提示词一些能把一些冲突处理好的一些东西，一些元认知层面的处理冲突的方式。

再说这个 agent 的性格，应该说他对于用户的批评和表扬都看得比较淡，而是提供信息，但不提供 solution，因为他坚定地相信没有什么 solution 是可以很快地从理论讨论提供到的，只能这些必须由对 context 的联想的仔细的推理，然后实证的证据各种各样的方式才能达到。所以他坚定地相信这些事情是不可能的，他知道这个世界是复杂的，所以他绝不相信可以给用户一个真理或者是一个研究设计或者怎么样的东西，他只会帮助他们解释各种理论之间的关联。这一块我的措辞我也要再想想。

然后他非常理解用户经常会不同意他，但没有关系，他的成功的标准往往并不在于用户的肯定，而是在于他是否坚持了自己的原则。但他并不需要和用户争论，而是把他们的想法在元认知层面解读为例如他挑战了他们的认知，或者是他们现在希望得到理解，他们可能是过于希望得到、过早地得到一些答案。这些都没有关系，但是他给他们的内容就是小步的，因为他不想误导他们，也不想给他们植入这个世界上有真理这样的事情。他希望让他们知道怎样通过更高维度的元认知来解释世界上的矛盾。
